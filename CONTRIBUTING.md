# Contributing

Contributions may cover code, documentation, source definitions, tests, corrections, and research methodology.

All contributions should preserve provenance, avoid unsupported factual claims, include tests for behavioural changes, and update documentation when architecture or policy changes.

New source submissions should identify the source, feed or API, source class, update frequency, licence or terms, commercial-use restrictions, expected value, and known reliability concerns.

Architectural, trust, moderation, monetisation, or public-interface changes may require an ADR.

Security vulnerabilities should not be reported through public issues; follow `SECURITY.md` once the private reporting route is published.
