# Initial Ingestion Scale

## MVP assumptions

- 50–100 active sources
- 300–1,500 raw items per day
- 50–300 clusters per day after deduplication
- 5–25 items per day receiving deeper analysis
- 3–10 items promoted into the main briefing

## Initial source mix

- 20–30 official provider/product sources
- 10–20 research and benchmark sources
- 10–20 repositories or release feeds
- 5–10 regulation and security sources
- 5–20 specialist publications and community feeds

## Freshness targets

- Breaking announcements: 1–3 hours
- GitHub releases: 1–6 hours
- Research feeds: 6–24 hours
- Pricing and API documentation: daily or on change
- Regulation: daily
- Model/provider records: weekly or after relevant releases
- Static methodology: monthly
