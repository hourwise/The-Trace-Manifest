          <section class="answer-hero" id="answer" data-od-id="direct-answer" aria-labelledby="answer-label">
            <p class="answer-label" id="answer-label">Direct current answer</p>
            <p class="answer-body">For tool-heavy agents, mid-tier closed models with recent list-price cuts often beat flagship reasoning models on cost per successful task. Open-weight APIs remain competitive when latency SLAs are flexible and you control the scaffold. There is no universal winner — value is workload-specific.</p>
            <div class="answer-why">
              <strong>Why (brief)</strong>
              Agent bills are dominated by output tokens and retries. Small quality gaps rarely justify multi-fold price differences unless failure cost is extreme. Scaffold efficiency can outweigh provider brand.
            </div>
          </section>

          <section class="rec-callout" id="recommendation" data-od-id="recommendation" aria-labelledby="rec-heading">
            <h2 id="rec-heading">Practical recommendation</h2>
            <p>Start with the current mid-tier closed model that has transparent tool docs and a recent price cut; parallel-test one open-weight API. Lock only after measuring success rate × cost on your scaffold. Revisit when list prices or model versions change.</p>
          </section>

            <p class="section-note">Each block is a different epistemic type. Colour is secondary; labels carry the meaning.</p>
            <div class="answer-stack">
              <article class="answer-block facts" data-od-id="facts">
                <h3>Facts</h3>
                <ul>
                  <li>Public list prices for major providers are documented; at least one frontier API adjusted rates recently.</li>
                  <li>Independent agent runs show scaffold choice moves scores more than some model swaps.</li>
                  <li>Open-weight hosted APIs publish lower token rates with different latency profiles.</li>
                </ul>
              </article>
              <article class="answer-block facts" data-od-id="supporting">
                <h3>Supporting evidence</h3>
                <p>Pricing tables; independent SWE-style cost write-ups; desk “Best value” methodology on the model index.</p>
              </article>
              <article class="answer-block conflict" data-od-id="conflicting">
                <h3>Conflicting evidence</h3>
                <p>Enterprise discounts, prompt caching, and batch APIs can invert public list-price rankings. Some vendors claim superior reliability that is not yet independently cost-normalised.</p>
              </article>
              <article class="answer-block vendor" data-od-id="vendor-claims">
                <h3>Vendor claims</h3>
                <p>Marketing often equates “best model” with “best value.” Treat as claims until matched-task cost studies exist for your scaffold.</p>
              </article>
              <article class="answer-block community" data-od-id="community">
                <h3>Community opinion</h3>
                <p>Practitioners frequently report overpaying for flagship models on routine agent loops; others report flagship savings via fewer retries. Both can be true for different harnesses.</p>
              </article>
              <article class="answer-block uncertain" data-od-id="uncertainty">
                <h3>Uncertainty</h3>
                <p>Your traffic mix, tool error rates, and caching strategy are unknown here. Confidence is capped until those are measured on a fixed internal harness.</p>
              </article>
              <article class="answer-block interp" data-od-id="interpretation">
                <h3>Interpretation</h3>
                <p>Prefer a short internal bake-off (50–100 tasks) across two mid-tier and one open-weight option before committing. Track cost per successful task, not tokens alone.</p>
              </article>

    } catch (_) {}
    return window.matchMedia("(prefers-color-scheme: light)").matches ? "light" : "dark";

    root.setAttribute("data-theme", theme);
    document.querySelectorAll("[data-theme-toggle]").forEach(function (btn) {

      var label = btn.querySelector("[data-theme-label]");
      if (label) label.textContent = theme === "light" ? "Light" : "Dark";
    });
  }

  function toggleTheme() {
    var next = root.getAttribute("data-theme") === "light" ? "dark" : "light";
    applyTheme(next);
    try { localStorage.setItem(THEME_KEY, next); } catch (_) {}
  }

  applyTheme(getPreferredTheme());

  document.addEventListener("click", function (e) {
    var t = e.target.closest("[data-theme-toggle]");

        c.classList.remove("active");
        c.setAttribute("aria-pressed", "false");

      var key = chip.getAttribute("data-filter");
      var target = group.getAttribute("data-filter-group");
      document.querySelectorAll("[data-filter-item]").forEach(function (item) {
        if (item.getAttribute("data-filter-scope") !== target) return;

        item.hidden = !show;
