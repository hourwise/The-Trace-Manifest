Indexing requirements

Every approved public page should have:

HTTP 200;
self-referencing canonical URL;
no noindex;

Sitemaps help search engines discover and revisit pages, while canonical URLs help consolidate duplicate or alternate versions. Search Console should be used to monitor indexing and errors.

AI-readable content

There is no dependable magic “AI scraping” tag that guarantees inclusion in AI answers.

The best practical approach is:

publish complete semantic HTML;
use short direct-answer sections;
use clear headings;
make claims and citations explicit;
expose original dates and update dates;
avoid burying key text in JavaScript widgets;
use consistent entity names;
provide public source links;
maintain stable URLs;
provide RSS or Atom feeds;
provide machine-readable JSON where appropriate;
permit only the crawlers you deliberately choose to permit.

Being crawlable does not guarantee an AI system will quote or cite the page. Different AI search systems choose different sources, and visibility can vary between ordinary search and generative answers. Recent research also suggests that generative search source selection differs materially from ordinary search results.
